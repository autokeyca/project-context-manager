**Technical Details**:

**1. onboard.py Script**:
- Interactive CLI with multiline input support
- Functions for each file type (CLAUDE.md, context.md, etc.)
- Collects project information via prompts
- Generates files from collected data
- Updates .gitignore automatically
- Supports `--skip-existing` flag for non-destructive updates
- Uses only Python stdlib (no external dependencies)

**2. File Generation**:
- `create_claude_md()` - Quick reference
- `create_context_md()` - Comprehensive documentation
- `create_tech_stack_md()` - Technical specifications
- `create_rules_md()` - Project-specific rules
- `create_next_steps_md()` - Prioritized roadmap
- `create_session_md()` - Ephemeral work state
- `create_journal_readme()` - Journal format guide
- `update_gitignore()` - Excludes session.md

**3. Project Questions**:
The script asks project-specific questions (NOT personal):
- Project name and purpose
- Current status
- Tech stack
- Architecture overview
- External integrations
- Deployment environment
- Known gotchas

**4. Directory Structure Created**:
```
<project>/.claude/
├── CLAUDE.md          # Quick reference
├── context.md         # Comprehensive docs
├── tech-stack.md      # Technical specs
├── rules.md           # Project rules
├── next-steps.md      # Roadmap
├── session.md         # Ephemeral (gitignored)
└── journal/           # Daily work logs
    ├── README.md
    └── YYYY-MM-DD.md
```

**Design Decisions**:
- **Standalone script** - No external dependencies for easy deployment
- **Interactive prompts** - Ensures complete documentation
- **Template-based** - Consistent structure across projects
- **Non-destructive** - Skip existing files option
- **Git-aware** - Updates .gitignore appropriately

**Usage Examples**:
```bash
# Basic onboarding
python3 onboard.py /path/to/project

# Interactive mode
python3 onboard.py

# Don't overwrite existing files
python3 onboard.py /path/to/project --skip-existing

# Batch onboard
for dir in /home/ja/projects/bots/*/; do
    python3 onboard.py "$dir" --skip-existing
done
```

**Integration**:
- Works with Elle's two-tier context system
- Global context: `~/.claude/.context/core/` (personal)
- Project context: `<project>/.claude/` (work-specific)
- Claude Code auto-detects and loads both
